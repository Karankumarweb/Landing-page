# Contributing to Landing Page

Thank you for your interest in contributing! We welcome contributions from everyone.

## 📋 Code of Conduct

This project adheres to the Contributor Covenant [code of conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## 🤝 How to Contribute

### Reporting Bugs

**Before submitting a bug report:**

1. Check the [issues](https://github.com/Karankumarweb/Landing-page/issues) to ensure it hasn't already been reported
2. Include as much detail as possible
3. Use a clear and descriptive title
4. Describe the exact steps to reproduce the problem
5. Provide specific examples to demonstrate the steps

**Create a bug report:**

1. Go to [New Issue](https://github.com/Karankumarweb/Landing-page/issues/new)
2. Select "🐛 Bug Report"
3. Fill in all the required information

### Suggesting Enhancements

**Before suggesting an enhancement:**

1. Check the [issues](https://github.com/Karankumarweb/Landing-page/issues) for existing suggestions
2. Use a clear and descriptive title
3. Provide a clear description of the enhancement
4. Include examples of how the feature would be used

**Create a feature request:**

1. Go to [New Issue](https://github.com/Karankumarweb/Landing-page/issues/new)
2. Select "✨ Feature Request"
3. Fill in all the required information

### Pull Requests

1. **Fork** the repository
2. **Create** your feature branch
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Make** your changes
4. **Test** your changes thoroughly
5. **Commit** your changes
   ```bash
   git commit -m 'Add amazing feature'
   ```
6. **Push** to the branch
   ```bash
   git push origin feature/amazing-feature
   ```
7. **Open** a Pull Request

## 📐 Development Guidelines

### HTML Guidelines

- Use semantic HTML5 elements
- Include proper alt text for images
- Use meaningful class and ID names
- Keep HTML organized and properly indented
- Add ARIA labels for accessibility

### CSS Guidelines

- Write clean, commented CSS
- Use consistent naming conventions (BEM)
- Group related styles together
- Use CSS variables for colors and spacing
- Ensure responsive design with media queries

### JavaScript Guidelines

- Write clean, readable code
- Add comments for complex logic
- Use meaningful variable names
- Avoid global variables
- Test across browsers

## 🧪 Testing

Before submitting a PR, ensure:

- [ ] Code is tested on Chrome, Firefox, Safari, and Edge
- [ ] Code is tested on mobile devices
- [ ] Accessibility is verified
- [ ] No console errors appear
- [ ] Page loads quickly

## 📱 Browser Testing

Test your changes on:

- Chrome (latest)
- Firefox (latest)
- Safari (latest)
- Edge (latest)
- Mobile browsers
